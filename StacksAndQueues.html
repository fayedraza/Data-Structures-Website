Stacks

Back to top

Stacks is a data structure where one can put an item on the top and then take off an item from the top.
Think about a stack of plates where you usually take off the top plate and if you wanted to put it back then you place it on the top of the stack.

Functions of Stacks

Stacks has many functions. We will be going over push, pop, isEmpty, size, and peek. These are all O(1) functions. We will go over each function.

Push

As you can see below, we have an empty stack as of now. What if we try to push a?

The letter a will be added on top of the stack as of now. What if we try to push b?

As you can see, b will be added on top of the stack. Basically, the push function adds an item onto the top of a stack just like our plates analogy.

Pop

As you can see below, we first have a stack with two elements. We called pop twice and the top element of the stack gets removed. Basically, the pop function deletes
the top of the stack. What if we try to call pop on an empty stack?

It will throw a NoSuchElementException which is why we should NEVER try to call pop on an empty stack.

isEmpty

As you can see below, an empty stack returns true and a non empty stack returns false. Basically, isEmpty returns true if the stack is empty and
false if the stack is not empty.

Size

As you can see, basically the number of elements of a stack is the value that will be returned.

Peek

As you can see, basically the element on the top gets returned and the stack stays the same.
Please note it will not work on empty stacks as shown on the image on the right.

LIFO

What is LIFO? It stands for Last In First Out which represents a stack as the latest element that was inserted will be the first element to be popped
if the pop function is called. It is really helps in terms of stacks memorization.

Queues

Back to top

Queues is a data structure where a line forms and the first one inserted will be deleted.
Think about a line at a ticket kiosk where the first one who came to the line will get the ticket.

Functions of Queues

Queues have many functions. We will be going over enqueue, dequeue, isEmpty, size, and peek. These are all O(1) functions. We will go over each function.

Enqueue

As you can see below, we have an empty queue as of now. What if we try to enqueue a?

a is added and the image on the right shows the result of when we enqueue b. The purpose of enqueue is to add elements.

Dequeue

As you can see below, we have a queue with two elements. What if we call dequeue?

In the first image the first element that was added, a, is deleted. And then b was deleted in the second image.
The purpose of dequeue is to delete the earliest added element. Now think what will happen if we call this method on an empty queue?

No Such Element Exception gets called which is why we should NEVER try to call dequeue on an empty queue.

isEmpty

As you can see below, an empty queue returns true and a non empty queue returns false. Basically, isEmpty returns true if the queue is empty and
false if the queue is not empty.

Size

As you can see, basically the number of elements of a queue is the value that will be returned.

Peek

As you can see, basically the earliest added element gets returned and the queue stays the same.
Please note it will not work on empty queue as shown on the image on the right.

FIFO

What is FIFO? It stands for First In First Out which represents a queue as the earliest element that was inserted will be the first element to be popped
if the dequeue function is called. It is really helps in terms of queues memorization.

Implementation of Queues Using Circular Linked Lists

We can implement queues using circular linked lists. We can use circular linked lists in enqueue, dequeue, isEmpty, size, and peek. Lets start with enqueue below.

In the enqueue function, if the rear is null/empty we can make a new node and point it to itself or if it is not null then set the next to the next of original rear.

The pesudocode of this method is on the left and the Java implementation is on the right.

In the field, we can declare the variable, size, in the field. In the enqueue function, it will increase by 1 and in the dequeue function, it will decrease by 1.

For the isEmpty method we can just check if the size is zero. If it is zero, then it is empty (return true).

For the peek method we can first check if it is null (empty) to see if we can return an element. If it does not, then we return the earliest element added to the queue using rear.next. The pesudocode of this method is on the left and the Java implementation is on the right.
