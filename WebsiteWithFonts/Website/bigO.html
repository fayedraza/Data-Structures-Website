    Finding the Big O

    Simplfying Big O Rules

    1. We will get rid of constants such as numbers like 1, 2, or etc, so 3n would turn to n.

    2. We will get rid of less dominant terms such as, if there was n^2 + n, we would get rid of n since it is less dominant. Also in cases like n + 3 it will be O(n) since 3 is just O(1) and it is less dominant than O(n).

    In this example, we got a runtime like 2n+1 and now we want to simplify it into Big O. First, we will get rid of the less dominant terms.
    O(2n + 1 )
    1 gets dropped since non dominant terms get dropped
    O(2n)
    2 gets dropped since constants get dropped
    Answer: O(n)

    In this example we got a runtime like n^2+n+1 and now we want to simplfy it into Big O. First we will get rid of the less dominant terms.
    O(n^2 + n + 1 )
    1 gets dropped since non dominant terms get dropped
    O(n^2 + n)
    n gets dropped since n is a less dominant term
    Answer: O(n^2)

    Big O Runtimes


    O(1) Runtime


    Examples of Java statements with O(1) runtime
  • int x = 7;
  • int b = 27*89;
  • String s = "hi";
  • System.out.println(a[0])

    • Example of an algorithm that has O(1) runtime (Pesudocode)
    READ number
    DISPLAY number * 3

    Despite the value of the number, it will always print that number multiplied by three meaning the time is constant which is why the runtime is O(1).

    Example of an algorithm that has O(1) runtime (Java)

    O(n) Runtime

    The O(n) runtime describes an algortihm that is porportional to its linear runtime.

    Classic examples of O(n) runtime
  • Tranversing through an array
  • Inserting to an arraylist (worst case)
  • In order traversal

    • Example of an algorithm that has O(n) runtime (Pesudocode)
    FOR each element in that array
    DISPLAY that element
    ENDFOR

    Since it will go through every element, so it depends upon how long the array is. If the length of an array is one, it will take quickly to go through the array. However, if the length is 1000, it will take a lot longer. In addition, it increases linearly as the length of an array increases by 1 its time increases by a bit too.

    Example of an algorithm that has O(n) runtime (Java)

    O(n^2) Runtime

    The O(n^2) runtime describes an algorithm that is porportional to its quadratic input of its size. For instance, an algorithm that has an input of 10 runs 100 times.

    Classic examples of O(n^2) runtime
  • Insertion Sort
  • Selection Sort
  • Quick Sort (Worst Case)

    • Example of an algorithm that has O(n^2) runtimne (Pesudocode)
    For each row
    For each element in that row
    DISPLAY that element
    ENDFOR
    ENDFOR

    Note: Array in the pseudocode input is n by n
    This is an example of a nested for loop. Since it is going through every element for each row, its run time is O (n^2). If the array length is three rows with three elements in each row then it will run nine times because 3^2 is 9.

    Example of an algorithm that has O(n^2) runtimne (Java)

    O(log n) Runtime

    The O(log n) runtime describes an algorithm that is proportional to its logarithm of that input. Just remember that in computer science when we talk about log n the base is always 2. If an input is 16 then it will run 4 times.

    Classic examples of O(log n) runtime
  • Binary Search
  • AVL Tree

    • Example of an algorithm that has O(log n) runtimne (Pesudocode)
    READ x
    WHILE x is greater than 0
    SET x as x divided by 2
    ENDWHILE

    This is a binary tree. Starting at the top which is 27 and as we choose to go left or right, we are eliminating the other half. For instance, if we choose to pick 40 instead of 14 then we will omit 14, 7, and 20. Then we go to 30 we omit 50. Now we have reached to the end of the tree and we hit 3 values. The floor of log base 2 of 7 is 3 hence this is why the Big O is O(log n).

    O(nlog n) Runtime

    The O(nlog n) runtime describes a an algorithm is proportional to the nlogn of the input.

    Classic examples of O(n logn) runtime
  • Merge Sort
  • Heap Sort

    • Example of an algorithm that has O(nlogn) runtime (Java). Also the image on the left shows how the Big O is O(nlogn). Basically, divide is log n and merge is n which is why the Big O turns out to be O(nlogn).

    O(2^n) Runtime

    The O(2^n) runtime describes an algorithm that is proportional of the 2^n of the input.

    Classic examples of O(2^n) runtime
  • Fibonacci sequence
  • Towers of Hanoi Problem

    • Example of an algorithm that has O(2^n) runtimne (Java). Basically, the output doubles each time. It doubles from 2 to 4 to 8. This is why the runtime is O(2^n).

    O(n!) Runtime

    The O(n!) runtime describes an algorithm that is proportional of the n! of the input.

    Example of an algorithm that has O(n!) runtime in Java. On the image on the right, if the input is 3 then for each element in the first loop of length 3 we traverse each element in length 2 and the traverse the final loop of length 1. So 3*2*1 = 6 which matches 3!.

    On another note, I want to talk about that an O(n) algorithm is proportional to its input. Meaning, lets say if an input array is a 2D Array and it runs O(n^2) but since they are proportional to each other, the Big O is O(n).

    More Information About Big O

    1. Sometimes there can be more than one input that affects the runtime, so the runtime can be like O(a + log b). In this case, a is the length of one array while b is the length of the other array. Both for loops do not affect each other so the runtime is O(a+b). Another example is finding the sum of a 2D rectangular array where one input is the number of rows and the other input is the length of each row (assuming that a and b does not have the same value). In this case, the Big O is O(ab).

    2. The Big O runtime O(n^2) can also have different exponents, such as O(n^3) which is slower than O(n^2) runtime. The higher the exponent the slower the runtime. An example is a triple nested for loop.

    3. The Big O runtime O(2^n) can also have a different base, such as 3^n which is slower than 2^n. The higher the base the slower the runtime.

    Typical memory used by primitive types

    Type Bytes
    Booelan 1
    Byte 1
    Char 2
    Int 4
    Float 4
    Long 8

    Examples of Space Complexity (Assume that we are using the chart above)

    Example 1: int x= 7;

    The space complexity is O(1) since an int is 4 bytes, so it will be O(4) which is O(1).

    Example 2: char a= 'x';

    The space complexity is O(1) since a char is 2 bytes. so it will be O(2) which is O(1).

    Example 3: An Array of Integers

    The space complexity is O(n) because we are talking about an integer array that holds many integers. Each integer is 4 bytes, so the space complexity of an array is O(4n) which is O(n).

    Example 4: Linked Lists

    The space complexity is O(n) because we are talking about a list that holds many nodes. n represents a node which means it is O(n).
