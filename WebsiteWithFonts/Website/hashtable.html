Computing the Load Factor

In order to computer the load factor, we have to divide the number of elements by the size of hashtable.

Every hash table has a treshold. The hash table current load facotr must be less than or equal to the load factor. So at the very bottom of the image the hash table's load factor surpasses the treshold. So think about what could be do to solve this problem. We will talk about it at the next image.

First we have double the size of the hash table. So the original size of the hash table is 2 will be doubled to 4. And also we have to rehash all of the original values. Also, in the mod function, we have to change it to 4 instead of 2 since 4 is the new size now. This process is called rehashing.

Now insert 14 into the hashtable and see if we need to rehash or not. Answer and/or explanation will be at the next image.

So after inserting 14, the load factor surpasses the treshold which means we need to rehash. We havae the double the size of the original hashtable and rehash all of the original elements.

Applications of Hash Tables: Unique Emails

Every email consists of a local name and a domain name, separated by the @ sign.
For example, in alice@leetcode.com, alice is the local name, and leetcode.com is the domain name.
Besides lowercase letters, these emails may contain '.'s or '+'s.
If you add periods ('.') between some characters in the local name part of an email address, mail sent there will be forwarded to the same address without dots in the local name. For example, "alice.z@leetcode.com" and "alicez@leetcode.com" forward to the same email address. (Note that this rule does not apply for domain names.)
If you add a plus ('+') in the local name, everything after the first plus sign will be ignored. This allows certain emails to be filtered, for example m.y+name@email.com will be forwarded to my@email.com. (Again, this rule does not apply for domain names.)
It is possible to use both of these rules at the same time.
Given a list of emails, we send one email to each address in the list. How many different addresses actually receive mails?


Example 1:
Input: ["test.email+alex@leetcode.com","test.e.mail+bob.cathy@leetcode.com","testemail+david@lee.tcode.com"]
Output: 2
Explanation: "testemail@leetcode.com" and "testemail@lee.tcode.com" actually receive emails

The reason why the Big O of insert into a hashtable is O(1) because after finding the index to insert we just insert it before the original linked list since we do not worry about making it sorted in this case. If it was sorted then it would be O(n) since we have to search through numbers to find the right place to insert. This is also why the Big O is O(n) for search and delete since we search through elements to locate or delete.
