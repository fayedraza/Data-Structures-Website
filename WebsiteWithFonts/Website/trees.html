Heap is like a complete binary tree and priority queue had a baby. Basically it is a tree where all of the nodes
are filled from left to right. And always the the highest parent is the minimum or maxium element of a tree.

Heap is like a complete binary tree and priority queue had a baby. Basically it is a tree where all of the nodes
are filled from left to right. And always the the highest parent is the minimum or maxium element of a tree.

As you can see, all of the nodes of a heap must be filled from left to right except for the last level. In the last level it does not have to be fully filled but if there are nodes then it should be filled from left to right.

There are two types of heaps: min heap and max heap. In the min heap the smallest element is on the top while in
the max heap the largest element is on the top.

Big O of Binary Search Trees

On average the Big O of going through a tree is O(log n) as we are eliminating paths. However, the worst case is O(n) as we will go through a skewed tree. Going down a skewed tree is just going through a linked list.

All of the tree traversals has a big o of O(n) because in the worst case we will go through n nodes.
